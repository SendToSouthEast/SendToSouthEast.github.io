QQ安卓端聊天记录数据分析 一

起因与现状

起因

起因呢……是我想备份和好朋友们的群聊聊天记录。其实聊天记录的文件信息并不多，不知道QQ是怎么动辄30G甚至更多……而且，时间一久，聊天记录就不让看了（指从群聊的查找聊天记录-日期）然而文件还没删，于是就开始了此项目。

现状

网络上有一些提取工具。如广为传播的付费软件——楼月以及各种杂七杂八的软件。这些软件不是要求付费，就是消息乱码、不可使用、无法导出，所以，就开始了此项目。

曙光

曙光是这个github项目。

GitHub - Yiyiyimu/QQ_History_Backup: QQ聊天记录备份导出，支持无密钥导出，图片导出。无需编译有GUI界面

数据库数据初步分析

QQ安卓端的数据库采用sqlite3，所以使用py的sqlite3库就很方便了。

QQ安卓端的数据库并非是全库加密，而是数据加密。

聊天记录的数据，存放于\com.tencent.mobileqq\databases(db)\QQ号.db

比较久远的聊天记录转移进了slowtable_QQ号.db

表单大纲

我使用的是SQLiteSpy软件来查看数据库。先打开QQ号.db

有好多好多张表。不要慌，很多是对我们没用的表，如动作QQ秀等乱七八糟的。不用理会。

有很多mr_Friend开头的表，推测这就是聊天记录信息了。

果然。这个表里面的东西，我们暂时略过，后面再做详细分析。我们先看表名：

mr_Friend_xxx_New。xxx正是对面QQ的md5值。（mr_Troop_xxx_New同理为群聊信息）

下面是这张表：Friends。

Friends表分析

这个表包含着你的好友和昵称、对方svip信息等等。我们需要uin（QQ号）name（昵称）。其它数据请自行研究。

数据：

列名 数据
uin QQ号
name 名称
remark 备注
mCompareSpell 用户名拼音-qq号
datetime 可能是加好友时间

这个表里还混杂着坦白说等等无关信息，我的方案是查看是否存在mr_Friend_xxx_New这张表来筛选。

TroopInfoV2表分析

和Friends表大同小异，存储群信息

列名 数据
troopuin 群组uin
troopname 群组名称
Administrator 管理员1QQ号|管理员2QQ号|…………
mCompareSpell 群拼音-qq号
troopCreateTime 创建时间
troopowneruin 群主QQ号

TroopMemberInfo表分析

群友信息表，所有的群友都在同一个表里

列名 数据
troopuin 群友所在群
memberuin 群友QQ
friendnick 用户名
troopnick 群昵称
join_time 入群时间
NewRealLevel 在群等级

真服了QQ，列名还有不同的方式……驼峰法、全小写、“_”间隔 一应俱全

mr_friend(trpoop)_XXXXXXX（QQ号MD5）_New表分析

列名 数据
msgtype 消息类型，下一篇重点
senderuin 发送者uin，需解密
msgData 消息内容，需解密
time 时间戳，不用解密
extStr 额外消息（属性等）需解密
